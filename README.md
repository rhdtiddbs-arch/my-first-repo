# 🐧 Linux Process Management Commands

리눅스 환경에서 프로세스를 모니터링하고 제어하기 위해 자주 사용하는 핵심 명령어 4가지(top, ps, jobs, kill)에 대한 정리입니다.

---

## 📊 명령어 한눈에 보기 (Summary)

| 명령어 | 모니터링 방식 | 대상 범위 | 주요 식별자 | 특징 |
| :--- | :--- | :--- | :--- | :--- |
| **top** | 실시간 (동적) | 시스템 전체 | PID (Process ID) | 리소스 점유율 중심의 실시간 화면 업데이트 |
| **ps** | 스냅샷 (정적) | 시스템 전체 | PID (Process ID) | 실행 시점의 고정된 상태 출력 (필터링에 유용) |
| **jobs** | 스냅샷 (정적) | 현재 터미널 세션 | 작업 번호 (%1, %2) | 백그라운드 및 일시 정지된 작업 관리 |
| **kill** | - | 특정 프로세스/작업 | PID 또는 작업 번호 | 프로세스에 시그널을 전달하여 종료 및 제어 |

---

## 🔍 명령어 상세 설명

### 1. 실시간 시스템 모니터링 top
시스템의 전반적인 상태와 실행 중인 프로세스 목록을 실시간으로 화면에 지속해서 업데이트하여 보여주는 **동적 모니터링 도구**입니다.

* **주요 특징**
  * CPU 및 메모리 사용량, 시스템 부하(Load Average) 등을 한눈에 파악할 수 있습니다.
  * 리소스 점유율이 높은 프로세스가 상단에 정렬되므로 시스템 병목 원인을 즉각 찾을 수 있습니다.
* **💡 유용한 단축키 (실행 상태에서 입력)**
  * q : top 화면 종료
  * M : 메모리(Memory) 사용량 순으로 정렬
  * P : CPU 사용량 순으로 정렬 (기본값)

---

### 2. 특정 시점의 프로세스 스냅샷 ps
명령어를 실행하는 그 순간, 시스템에 존재하는 프로세스의 고정된 상태를 보여주는 **정적 도구**입니다.

* **주요 특징**
  * top과 달리 실시간으로 변하지 않고 명령어 실행 당시의 스냅샷을 텍스트로 출력합니다.
  * 보통 시스템 전체 프로세스를 상세히 보기 위해 옵션을 자주 조합하여 사용합니다.
* **주요 옵션 조합**
  * ps -ef : 시스템의 모든 프로세스를 풀 포맷(Full Format)으로 출력
  * ps aux : BSD 계정 스타일로 CPU, 메모리 사용량까지 포함하여 상세 출력
* **💡 실무 팁**
  * 출력량이 매우 많기 때문에, 파이프(|)와 grep 명령어를 조합하여 특정 프로세스를 필터링하는 경우가 많습니다.

### 3. 현재 터미널 세션의 작업 관리 jobs
현재 사용자가 접속 중인 터미널 세션 내부에서 제어하고 있는 백그라운드 작업 목록을 보여줍니다.

* **주요 특징**
  * 명령어 뒤에 앤드 기호(&)를 붙여 백그라운드로 실행하거나, 실행 중인 명령을 Ctrl + Z로 일시 정지시켜 백그라운드로 보낸 작업들을 관리합니다.
  * 작업 번호(Job ID)와 Running(실행 중), Stopped(일시 정지) 같은 상태를 표시합니다.
* **⚠️ 주의 사항**
  * 시스템 전체 프로세스를 보여주는 ps나 top과 달리, 자신이 명령어를 내린 **현재 터미널 내의 작업만** 볼 수 있습니다.
* **💡 실무 팁**
  * 백그라운드로 전환된 작업을 다시 포그라운드(화면 앞)로 가져오려면 fg %작업번호 명령어를 함께 사용합니다.

### 4. 프로세스 강제 종료 및 시그널 전달 kill
실행 중인 특정 프로세스나 작업에 시그널(Signal)을 보내 제어하거나 종료할 때 사용합니다.

* **주요 특징**
  * 본질은 프로세스를 단순히 종료하는 것이 아니라, 프로세스에 특정 **명령(시그널)을 전달**하는 역할입니다.
  * 앞서 ps나 jobs 명령어로 확인한 식별자 정보를 기반으로 프로세스를 제어합니다.
* **주요 시그널 조합**
  * kill -15 [PID] : 기본값(SIGTERM). 프로세스가 안전하게 리소스를 정리하고 종료되도록 요청합니다.
  * kill -9 [PID] : 프로세스가 종료 요청에 응답하지 않을 때, OS가 즉시 강제 종료(SIGKILL)합니다.
* **💡 실무 팁**
  * ps로 확인한 ID는 그냥 숫자만 입력하고, jobs로 확인한 작업 번호를 지정할 때는 숫자 앞에 반드시 퍼센트(%) 기호를 붙여야 합니다.
